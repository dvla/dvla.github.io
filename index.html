Open Source at the Driver and Vehicle Licensing Agency

This page is aimed at software developers. To use online DVLA services, please go to https://www.gov.uk instead.

Why Open Source our code?

Over the last few years, open source has taken the software industry by storm. The UK Government now has clear guidance to open source software, and the DVLA are moving toward a open source model for its new digital transformation programme.

The DVLA is developing its new government services on top of open source technologies and libraries, including Linux, Scala, Java, Python, Ruby. It is clear from the functionality that we get from open source code, that open source works. Therefore, it is healthy for a heavy user of open source to be part of the community that we take part in.

Exemplar Programmes within the DVLA

The DVLA is undergoing a digital transformation, and Digital Exemplars are the beginning of that journey. Exemplars will be open sourcing their code, and the DVLA Github organisation contains the current ones.

Vehicles Management Exemplars

Vehicles https://www.gov.uk/transformation/manage-vehicle
Personalised Registrations https://www.gov.uk/transformation/personalised-number-plates

The vehicles Exemplar has 34 git repositories, and currently have open sourced 6 of those. Of the code open sourced, there are 22260 lines of Scala code, some Javascript, Ruby and Python. Our projects are as follows:

  • https://github.com/dvla/vehicles-online is a web application which allows motor traders to dispose their vehicles to the trade. It is written in Scala and the Play framework
  • https://github.com/dvla/vrm-retention-online is a web application which allows personalised registrations to be taken off cars and retained by the owner. It is written in Scala and Play.
  • https://github.com/dvla/os-address-lookup is micro-service for looking up addresses in the Ordnance Survey service using a postcode or a UPRN. Requires purchasing an API key from Ordnance Survey
  • https://github.com/dvla/proxypy is a distributed, highly available rate-limited proxy server written in Python. It is used for throttling HTTP requests within our micro service architecture
  • https://github.com/dvla/vehicles-services-common contains common code used across the vehicle micro-services, which are written in Scala and Spray
  • https://github.com/dvla/vcloud-management-tools is one of a number of build automation libraries we have. It is a vcloud-rest wrapper that allows us to interact with vmware vcloud providers such as skyscape

Will the code be useful for other people?

Much of this code is specific to government services, and therefore will see reuse within the government. Some utility libraries, such as proxypy might be useful for a more general audience. In some cases, the code might be useful for developers to see how another team has approached solved a specific problem, what coding styles team use, etc.

The important point is that the code available now is the start of the journey. More code will be appearing, evolving, and hopefully giving back to the open source community. Getting our developers in the mindset of community development is important.

Why can't all the code be open sourced?

There is a significant proportion of our code that cannot be open sourced. Exclusions to the open source process might be based around the following:

  • The DVLA has to maintain the integrity of its systems and data. Some information that is deemed useful to attackers & fraudsters will remain closed source
  • The DVLA have line of business systems that have been processing transactions for many years. They were not build with an open source ethos from the beginning. These existing systems are not run or maintained by the new project teams. It is unclear whether this code will ever transition to open source for contractual reasons and the effort associated with splitting the system into open and closed parts. Therefore any of the new code that exposes interfaces to legacy systems, such as API contracts (e.g. WSDL), will remain closed source. As the IT estate gets refreshed, closed source will represent a decreasing proportion of the code owned and maintained by the DVLA.

Although much of the code is not open sourced, we try to ensure that the code should compile and tests can be run. We use Travis CI to verify that the code builds from Github. The status of Travis CI should be viewable from the README files from the repositories that have it enabled.

Are you using Github as your primary code repository?

No, the DVLA carries out development within protected repositories in a government data centre in UK territory. This protects the integrity of the source code, whilst publishing to github provides the best place for the open source community to view it.

We review code before it is pushed out into the public domain, to ensure that it doesn't contain sensitive information.

Can I contribute to these repositories?

Yes, absolutely. We can accept pull requests in Gitlab and merge them with our protected repository, after a review has taken place. DVLA to retain the right to not accept changes if they alter the code in a way which is incompatible with its goals.

What licence is the code?

The MIT licence is used for most code, which is a very permissive licence without any copyleft requirements like the GPL. Please check the root of the repository you intent to use for its specific licence.
